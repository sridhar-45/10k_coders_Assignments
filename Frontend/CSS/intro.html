Understanding CSS

What it is, why we use it, who invented it, versions, responsive design, layouts, and styling methods.

What is CSS?

CSS stands for Cascading Style Sheets. It is a style sheet language used to describe the presentation (look and feel) of a document written in HTML. It controls layout, colors, fonts, spacing, animations, responsiveness, etc.

Simply: HTML is the structure (skeleton), CSS is the style (clothes/skin).

Why is CSS used?

  • Separation of content & design → HTML handles content, CSS handles design.
  • Reusability → One CSS file can style multiple HTML pages.
  • Consistency → Keeps the look & feel uniform across a website.
  • Responsive design → Makes websites adapt to different devices (PC, Tablet, Mobile).
  • Faster development → Change style in one place → reflects everywhere.

Who invented CSS?

Håkon Wium Lie and Bert bose invented CSS in 1994

Present Version of CSS

The latest stable version is CSS3 (introduced between 1999–2011). CSS3 is modular, divided into parts like Selectors, Backgrounds, Animations, Flexbox, and Grid. Work continues with new features (sometimes informally called CSS4), but there is no single official CSS4 version.

Responsive Web Design (RWD)

Responsive design ensures websites adjust automatically to desktops, tablets, and phones. Achieved using:

  • Media Queries (@media screen and (max-width: 768px) { ... })
  • Flexible layouts (Flexbox, CSS Grid)
  • Fluid images & percentages instead of fixed pixels

CSS Layouts

  1. Normal Flow (default)
  2. Box Model (margin, border, padding, content)
  3. Positioning (static, relative, absolute, fixed, sticky)
  4. Flexbox (1D layout - row or column)
  5. CSS Grid (2D layout - rows + columns)
  6. Responsive layouts with media queries

Styling Methods in CSS

1. Inline CSS

Written inside the HTML tag using the style attribute. Useful for quick testing, but not recommended for large projects.

2. Internal CSS

Defined inside a style tag within the HTML file (inside head). Good for small projects or single-page sites.

3. External CSS

Written in a separate css file and linked with link. Best practice for large websites and reusability.
